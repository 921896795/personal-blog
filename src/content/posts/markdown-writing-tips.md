Markdown 之所以适合长期写作，是因为它足够简单，不会被排版分心。

## 标题层级要克制

一篇文档最多用到三级标题。如果发现自己在用四级、五级标题，通常意味着文章结构需要拆分。

## 列表比段落更适合说明

当你在解释步骤、对比选项、罗列要点时，列表比大段文字更容易被阅读和记住。

## 代码块要有语言标记

给代码块加上语言标识，不仅能让编辑器高亮，也能让静态站点生成器正确地做语法高亮。

```python
def hello():
    print("Hello, blog!")
```

## 图片要写 alt

```md
![架构图](/images/arch.png)
```

alt 文本不只是为了无障碍，也是给搜索引擎的上下文。

## 结语

好的写作工具应该让你更专注于内容本身，而不是排版细节。Markdown 做到了这一点。
